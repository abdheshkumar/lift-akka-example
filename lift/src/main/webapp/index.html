Welcome to the Liftweb/Akka example project!

This project was presented at the Scala LiftOff Conference 2011 in London.
It was part of my Talk Liftweb in the Real World. A hyper-polygot's tale.

Duplicate a file (on a remote host)

Choose a file that you want a backend Actor to duplicate with rsync. The parsed output will be "streamed" via Comet and produce a nice jQuery-UI Progressbar. It will only show files larger than 200MB which reside within the directory set as filecopy.path in .props. This works with either a LiftActor or an AkkaActor (both included in source).

How does it all work?

We are combining the power of Akka with Liftweb's ability to perform Comet-style updates on the client's browser. None of this is scheduled AJAX, it's all Comet. The copy-process in the background uses rsync, so make sure it is in PATH.
